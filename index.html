Critical Concepts

Lesson 1: The Basics of the Web and HTML

The Basics of the Web:
  • World Wide Web or "Web" is a collection of HTML documents.
  • HTML is Hyper Text Markup Language, which is the basis for almost every webpage. Hyperlinks or "links" glue these pages together.
  • Web Browsers such as Internet Explorer, Safari, Firefox, etc. makes requests via the Internet to servers.
  • HTTP is the protocol used by web browsers and servers to communicate with the Web.
  • Servers are computers that host files that interact with internet browsers over the Web.
What is HTML?
  • Text content (what you see)
  • Markup (what that content looks like or how it is arranged)
  • References to other documents (i.e. images and videos)
  • Links to other pages (i.e. www.udacity.com)
  • Elements in HTML are made up of opening tags, contents, and closing tags.
  • Common stylistic tags used in HTML are the bold tags and "em" tags. The bold tags does just what it says, it makes text bold. The "em" tag adds emphasis or italicizes words.
  • HTML attributes are a tag that equals some sort of value. One type of HTML attribute is the anchor tag, which is used to create links.
  • All text in HTML is rendered as one line, so special tags must be used such as the paragraph tag or the "br" tag to break up text.
  • There are two different types of elements in HTML -- inline and block. Inline merely formats text. Block actually creates a box with formatting, width, etc.
Why Computers are Stupid

Computers interpret instructions and code literally. Therefore, any mistake can cause huge mistake for a developer, as as computers are unforgiving. Developers should also be careful about repeating themselves to prevent errors.

Lesson 2: Creating a Structured Document with HTML

  • Developing a Page -- All HTML documents are a series of boxes and as important as the coding of the page is its layout and planning of text, images, and item placement.
  • HTML is Organized in a Tree-like Structure whereby the elements can have other elements subsumed inside of them (and often do). It is useful to outline these elements like a family tree with the various children or splinters of one element coming off another element to understand and determine the relationship from the first element. In a Internet Browser, this will show up as series of nested boxes within boxes.
  • Google Chrome's Developer Tools is a very effective tool enabling software developers to view this tree-like structure and nested relationships and understand how the browser interacts with the code and styling.
  • Text Editors such as Sublime Text or CoffeeCub HTML makes programmers lives easier by automatically creating HTML tags for coders and fixing errors.

Lesson 3: Cascading Style Sheets & Avoiding Repetition

  • Avoiding Repetition is important because when developers avoid repetition they avoid errors. By avoiding repetition through the use of copying and pasting certain style attributes into many HTML elements or rewriting the same code, many times this causes errors. The better approach for style elements that repeat themselves is the use of Cascading Style Sheets discussed below.
  • CSS or Cascading Style Sheets is a language used to describe the look and format of a document created using a markup language, such as HTML.
  • Inheritance is a key concept in CSS. Like ancestors pass on traits to their relatives, Cascading Style Sheet elements pass on traits to elements further down the document tree. This generally applies to text related properties. Properties related to boxes generally cannot be inherited. Check out this awesome article on CSS Inheritance at Wikipedia.
